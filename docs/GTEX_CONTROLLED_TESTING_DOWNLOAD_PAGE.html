Controlled Testing Status

This build is intended for a limited tester group. It is suitable for install, launch, smoke testing, and issue capture during the RC window.

  • Not for public store release or open distribution.
  • Use only approved test accounts, data, and reporting channels.
  • Expect rapid iteration while RC feedback is being collected.

What’s Included In This RC

  • Current GTEX Android release candidate package for controlled install and launch validation.
  • Approved internal testing pass for RC smoke coverage and issue reporting.
  • Packaging and runtime checks needed to confirm the build is shareable with testers.
  • Placeholder space for final release notes if the team wants to list approved flows here.

Known Limitations

  • This is a controlled RC build, so behavior may still change before general release.
  • The APK download section is currently a placeholder until the final hosted link is added.
  • Testing coverage is limited to approved internal scenarios, devices, and accounts.
  • Non-critical defects may remain open while the team evaluates final RC feedback.

Tester Instructions

  1. Download the APK from the approved internal link.
  2. Install it on the assigned Android device and allow manual install if prompted.
  3. Launch GTEX and complete the agreed smoke-test scenarios for this RC.
  4. Record the device model, Android version, app build reference, and time of issue if anything fails.
  5. Capture screenshots or screen recordings for anything user-visible.

How To Report Issues

Send findings through the designated internal reporting channel or tracker. If the final channel is not yet set, replace this section with the approved Slack, email, Jira, or form link.

  • Summary: short title of the problem.
  • Severity: blocker, major, minor, or cosmetic.
  • Device: phone model and Android version.
  • Build: GTEX RC APK reference used for the test.
  • Steps: exact reproduction steps.
  • Expected vs actual: what should have happened and what happened instead.
  • Evidence: screenshots, recording, logs, or timestamps.
